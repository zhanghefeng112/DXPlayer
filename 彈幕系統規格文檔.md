## 1. 系統概述

### 1.1 背景

彈幕系統是一種在影片播放時，讓使用者可以發送滾動文字評論的互動功能。彈幕會從畫面右側飛入，向左滾動至畫面左側消失，讓觀眾可以即時分享觀影感受。

### 1.2 目標

- 提供流暢的彈幕顯示體驗，支援多行同時滾動
- 支援彈幕的發送與接收機制
- 提供完整的彈幕設定選項（透明度、顯示行數、字體大小、滾動速度）
- 整合到現有的 DXPlayerSDK 框架中
- 支援靈活的資料來源架構

### 1.3 適用範圍

- iOS 平台（最低版本：iOS 12.0）
- 整合於 DXPlayerSDK 框架
- 適用於所有使用 FSPlayer 的影片播放場景

### 1.4 核心特性

1. **多行顯示**: 支援 1-5 行彈幕同時顯示
2. **碰撞檢測**: 智慧分配彈幕軌道，避免重疊
3. **高度可配置**: 透明度、字體、速度均可調整
4. **效能優化**: 支援彈幕池複用，減少記憶體開銷
5. **時間同步**: 彈幕與影片時間軸精準同步
6. **靈活架構**: 支援多種資料來源模式

## 2. 功能需求

### 2.1 彈幕顯示

#### 2.1.1 基本顯示

- **顯示區域**: 影片畫面上方區域
- **顯示行數**: 可配置 1-5 行（預設 3 行）
- **滾動方向**: 從右向左滾動
- **顯示範圍**: 僅在安全區域內顯示（避免被瀏海或導航欄遮擋）

#### 2.1.2 顯示邏輯

```
當彈幕到達時：
1. 檢查彈幕開關狀態
   - 若關閉：不顯示
   - 若開啟：繼續步驟 2

2. 根據當前配置的行數，選擇可用軌道
   - 優先選擇空閒軌道
   - 若無空閒軌道，選擇最早結束的軌道

3. 計算彈幕起始位置（畫面右側外）

4. 根據速度設定計算動畫時長

5. 啟動滾動動畫（從右至左）

6. 彈幕完全離開畫面後，回收至彈幕池
```

#### 2.1.3 顯示優先級

- **同一時間點的多條彈幕**: 按接收順序顯示
- **軌道衝突**: 使用碰撞檢測算法分配軌道
- **效能保護**: 當彈幕密度過高時，自動丟棄部分彈幕

### 2.2 彈幕發送

#### 2.2.1 發送入口

- **位置**: 固定於控制列（Control Bar）
- **觸發方式**: 點擊「彈幕」按鈕顯示輸入框
- **輸入限制**:
  - 最大字數：100 字
  - 禁止空白彈幕
  - 表情符號支援：✅

#### 2.2.2 發送流程

```
1. 使用者點擊「彈幕」按鈕
2. 顯示輸入框（鍵盤彈出）
3. 使用者輸入文字
4. 點擊「發送」按鈕
5. 驗證輸入內容
   - 長度檢查
   - 敏感詞過濾（可選）
6. 呼叫發送 API
7. 發送成功後：
   - 立即在本地顯示該彈幕（無需等待伺服器返回）
   - 關閉輸入框
   - 清空輸入內容
8. 發送失敗後：
   - 顯示錯誤提示
   - 保留輸入內容
```

#### 2.2.3 發送限制

- **頻率限制**: 5 秒內最多發送 1 條（可配置）
- **長度限制**: 1-100 字
- **登入要求**: 需要登入後才能發送（可配置）

### 2.3 彈幕設定

#### 2.3.1 設定選項

根據使用者提供的截圖，設定面板包含以下選項：

| 設定項目 | 類型 | 範圍 | 預設值 | 說明 |
|---------|------|------|--------|------|
| 透明度 | 滑桿 | 0-100% | 100% | 控制彈幕文字的不透明度 |
| 顯示行數 | 滑桿 | 1-5 行 | 3 行 | 控制同時顯示的彈幕軌道數量 |
| 字體大小 | 滑桿 | 小/標準/大 | 標準 | 控制彈幕文字大小 |
| 速度 | 滑桿 | 慢/普通/快 | 普通 | 控制彈幕滾動速度 |

#### 2.3.2 字體大小對應

- **小**: 12pt
- **標準**: 16pt
- **大**: 20pt

#### 2.3.3 速度對應

- **慢**: 15 秒/屏
- **普通**: 10 秒/屏
- **快**: 6 秒/屏

#### 2.3.4 設定持久化

- 所有設定選項應儲存至本地（UserDefaults）
- 應用啟動時自動載入上次設定
- 設定變更應立即生效

### 2.4 彈幕開關

#### 2.4.1 開關位置

- **位置**: 控制列（Control Bar）
- **狀態指示**:
  - 開啟：圖示高亮
  - 關閉：圖示灰色

#### 2.4.2 開關邏輯

```
開關關閉時：
- 停止接收新彈幕
- 已顯示的彈幕繼續完成動畫後消失
- 不清空彈幕佇列（待開啟時繼續顯示）

開關開啟時：
- 恢復接收彈幕
- 從佇列中取出彈幕繼續顯示
```

### 2.5 資料來源模式

系統支援多種彈幕資料來源模式，由整合方決定使用哪種：

#### 2.5.1 模式 A：時間軸預載模式

```
適用場景: 點播影片（VOD）
資料特性: 彈幕與影片時間軸綁定
載入方式: 播放前批量載入

流程:
1. 影片載入時，從伺服器獲取該影片的所有彈幕資料
2. 資料格式: [{time: 5.2, text: "好看"}, {time: 10.5, text: "精彩"}]
3. 根據當前播放時間，從資料集中篩選需要顯示的彈幕
4. 當播放時間到達彈幕的 time 時，觸發顯示

優點:
- 彈幕與影片完美同步
- 支援快進/快退精準定位
- 離線緩存友好

缺點:
- 初始載入時間較長（影片彈幕量大時）
- 無法實現真正的「即時互動」
```

#### 2.5.2 模式 B：WebSocket 即時推送模式

```
適用場景: 直播場景
資料特性: 彈幕即時產生，與時間軸無關
載入方式: WebSocket 即時推送

流程:
1. 播放器啟動時，建立 WebSocket 連線
2. 伺服器即時推送彈幕訊息
3. 收到訊息後立即顯示（無需時間軸對應）
4. 使用者發送的彈幕透過 WebSocket 送出

優點:
- 真正的即時互動
- 適合直播場景
- 無需預載

缺點:
- 需要維持 WebSocket 連線
- 無法重播（除非錄製）
- 網路依賴性強
```

#### 2.5.3 模式 C：混合模式

```
適用場景: 點播 + 即時評論
資料特性: 歷史彈幕 + 即時彈幕
載入方式: 預載 + 推送

流程:
1. 播放前載入歷史彈幕（時間軸綁定）
2. 同時建立 WebSocket 連線接收即時彈幕
3. 歷史彈幕按時間軸顯示
4. 即時彈幕立即顯示（無論當前播放時間）

優點:
- 兼具歷史回顧與即時互動
- 靈活性最高

缺點:
- 實現複雜度較高
- 需要區分兩種彈幕來源
```

#### 2.5.4 模式 D：本地模式（測試/演示）

```
適用場景: 開發測試、演示
資料特性: 本地產生的模擬資料
載入方式: 無需網路

流程:
1. 使用本地資料源（JSON 文件或記憶體陣列）
2. 模擬彈幕發送與接收
3. 用於測試 UI 與動畫效果

優點:
- 無需後端支援
- 適合快速開發與測試

缺點:
- 僅供開發使用
```

### 4.2 DanmakuSettings

彈幕設定配置。

```swift
/// 彈幕設定
public struct DanmakuSettings {
    /// 透明度（0.0 - 1.0）
    public var opacity: CGFloat

    /// 顯示行數（1 - 5）
    public var displayLines: Int

    /// 字體大小
    public var fontSize: DanmakuFontSize

    /// 滾動速度
    public var speed: DanmakuSpeed

    /// 是否啟用彈幕
    public var isEnabled: Bool

    /// 預設值
    public static var `default`: DanmakuSettings {
        return DanmakuSettings(
            opacity: 1.0,
            displayLines: 3,
            fontSize: .standard,
            speed: .normal,
            isEnabled: true
        )
    }

    /// 初始化
    public init(
        opacity: CGFloat = 1.0,
        displayLines: Int = 3,
        fontSize: DanmakuFontSize = .standard,
        speed: DanmakuSpeed = .normal,
        isEnabled: Bool = true
    ) {
        self.opacity = max(0.0, min(1.0, opacity))
        self.displayLines = max(1, min(5, displayLines))
        self.fontSize = fontSize
        self.speed = speed
        self.isEnabled = isEnabled
    }
}

/// 字體大小
public enum DanmakuFontSize: Int, CaseIterable {
    case small = 0      // 12pt
    case standard = 1   // 16pt
    case large = 2      // 20pt

    public var points: CGFloat {
        switch self {
        case .small: return 12.0
        case .standard: return 16.0
        case .large: return 20.0
        }
    }

    public var displayName: String {
```

#### 5.2.2 WebSocketDanmakuDataSource（即時模式）

```swift
/// WebSocket 即時模式資料來源
public class WebSocketDanmakuDataSource: DanmakuDataSource {
    public var sourceType: DanmakuSourceType { return .realtime }
    public weak var delegate: DanmakuDataSourceDelegate?

    private var webSocket: URLSessionWebSocketTask?
    private var roomId: String?
    private var connectionState: DanmakuConnectionState = .disconnected {
        didSet {
            delegate?.dataSource(self, didChangeConnectionState: connectionState)
        }
    }

    public init() {}

    public func start(context: [String: Any]) {
        guard let roomId = context["roomId"] as? String else {
            delegate?.dataSource(self, didFailWithError: NSError(domain: "DanmakuDataSource", code: -1, userInfo: [NSLocalizedDescriptionKey: "缺少 roomId"]))
            return
        }

        self.roomId = roomId
        connectWebSocket()
    }

    public func stop() {
        webSocket?.cancel(with: .goingAway, reason: nil)
        webSocket = nil
        connectionState = .disconnected
    }

    public func send(text: String, completion: @escaping (Result<Void, Error>) -> Void) {
        guard connectionState == .connected else {
            completion(.failure(NSError(domain: "DanmakuDataSource", code: -2, userInfo: [NSLocalizedDescriptionKey: "WebSocket 未連線"])))
            return
        }

        let message = ["type": "danmaku", "text": text]
        guard let jsonData = try? JSONSerialization.data(withJSONObject: message),
              let jsonString = String(data: jsonData, encoding: .utf8) else {
            completion(.failure(NSError(domain: "DanmakuDataSource", code: -3, userInfo: [NSLocalizedDescriptionKey: "訊息編碼失敗"])))
            return
        }

        webSocket?.send(.string(jsonString)) { error in
            if let error = error {
                completion(.failure(error))
            } else {
                completion(.success(()))
            }
        }
    }

    public func fetchDanmaku(from startTime: TimeInterval, to endTime: TimeInterval, completion: @escaping (Result<[DanmakuItem], Error>) -> Void) {
        // 即時模式不支援此方法
        completion(.failure(NSError(domain: "DanmakuDataSource", code: -4, userInfo: [NSLocalizedDescriptionKey: "即時模式不支援時間範圍查詢"])))
    }

    // MARK: - 私有方法

    private func connectWebSocket() {
        guard let roomId = roomId else { return }

        connectionState = .connecting

        let url = URL(string: "wss://example.com/danmaku/\(roomId)")!
        webSocket = URLSession.shared.webSocketTask(with: url)
        webSocket?.resume()

        connectionState = .connected

        // 開始接收訊息
        receiveMessage()
    }

    private func receiveMessage() {
        webSocket?.receive { [weak self] result in
            guard let self = self else { return }

            switch result {
            case .success(let message):
                switch message {
                case .string(let text):
                    self.handleMessage(text: text)
                case .data(let data):
                    if let text = String(data: data, encoding: .utf8) {
                        self.handleMessage(text: text)
                    }
                @unknown default:
                    break
                }

                // 繼續接收下一條訊息
                self.receiveMessage()

            case .failure(let error):
                self.delegate?.dataSource(self, didFailWithError: error)
                self.connectionState = .disconnected
            }
        }
    }

    private func handleMessage(text: String) {
        guard let data = text.data(using: .utf8),
              let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any],
              let type = json["type"] as? String,
              type == "danmaku",
              let content = json["text"] as? String else {
            return
        }

        let danmaku = DanmakuItem(
            id: json["id"] as? String ?? UUID().uuidString,
            text: content,
            userId: json["userId"] as? String,
            userName: json["userName"] as? String
        )

        delegate?.dataSource(self, didReceive: danmaku)
    }
}
```

## 8. 實現細節

### 8.1 軌道分配算法

#### 8.1.1 核心邏輯

```swift
class DanmakuOverlay: UIView {
    // 軌道資訊
    private var tracks: [DanmakuTrack] = []

    // 初始化軌道
    private func setupTracks(count: Int) {
        tracks = (0..<count).map { DanmakuTrack(index: $0) }
    }

    // 分配軌道
    private func allocateTrack(for danmaku: DanmakuItem) -> Int? {
        let currentTime = CACurrentMediaTime()
        let danmakuWidth = estimateWidth(for: danmaku)
        let screenWidth = bounds.width
        let speed = currentSettings.speed

        // 策略 1: 優先選擇空閒軌道
        for track in tracks {
            if track.danmakuViews.isEmpty {
                return track.index
            }
        }

        // 策略 2: 選擇可以放置的軌道（通過碰撞檢測）
        for track in tracks {
            if track.canPlace(at: currentTime, danmakuWidth: danmakuWidth, screenWidth: screenWidth, speed: speed) {
                return track.index
            }
        }

        // 策略 3: 無可用軌道，返回 nil（丟棄或等待）
        return nil
    }

    // 估算彈幕寬度
    private func estimateWidth(for danmaku: DanmakuItem) -> CGFloat {
        let fontSize = danmaku.fontSize ?? currentSettings.fontSize.points
        let font = UIFont.systemFont(ofSize: fontSize, weight: .medium)
        let attributes = [NSAttributedString.Key.font: font]
        let textSize = (danmaku.text as NSString).size(withAttributes: attributes)
        return textSize.width + 16 // 加上內距
    }
}
```

#### 8.1.2 碰撞檢測

```swift
extension DanmakuTrack {
    func canPlace(at currentTime: TimeInterval, danmakuWidth: CGFloat, screenWidth: CGFloat, speed: DanmakuSpeed) -> Bool {
        guard let lastView = danmakuViews.last else {
            return true // 軌道為空，可以放置
        }

        // 計算最後一條彈幕的當前位置
        let animationProgress = lastView.layer.presentation()?.frame.origin.x ?? lastView.frame.origin.x

        // 最小間距
        let requiredGap: CGFloat = 50

        // 檢查最後一條彈幕是否已經移出足夠距離
        // 即：最後一條彈幕的尾部已經離開畫面右側足夠距離
        return animationProgress < screenWidth - requiredGap
    }
}
```

### 8.2 彈幕池複用機制

為了減少記憶體開銷和視圖創建開銷，使用對象池模式複用彈幕視圖。

```swift
class DanmakuOverlay: UIView {
    // 彈幕池
    private var danmakuPool: [DanmakuLabel] = []
    private let maxPoolSize = 50 // 最多緩存 50 個視圖

    // 從池中取得彈幕視圖
    private func dequeueDanmakuView() -> DanmakuLabel {
        if let view = danmakuPool.popLast() {
            // 複用現有視圖
            return view
        } else {
            // 創建新視圖
            return DanmakuLabel()
        }
    }

    // 回收彈幕視圖到池中
    private func recycleDanmakuView(_ view: DanmakuLabel) {
        // 停止動畫
        view.layer.removeAllAnimations()

        // 從父視圖移除
        view.removeFromSuperview()

        // 重置狀態
        view.text = nil
        view.alpha = 1.0

        // 加入池中（如果未滿）
        if danmakuPool.count < maxPoolSize {
            danmakuPool.append(view)
        }
    }

    // 清空池（記憶體警告時呼叫）
    private func clearPool() {
        danmakuPool.removeAll()
    }
}
```

### 8.3 效能優化

#### 8.3.1 彈幕密度控制

當彈幕密度過高時，自動丟棄部分彈幕以保持流暢性。

```swift
class DanmakuManager {
    private var danmakuQueue: [DanmakuItem] = []
    private let maxQueueSize = 100 // 最多排隊 100 條

    func enqueueDanmaku(_ danmaku: DanmakuItem) {
        // 如果佇列已滿，丟棄最舊的彈幕
        if danmakuQueue.count >= maxQueueSize {
            danmakuQueue.removeFirst()
            DXPlayerLogger.warning("⚠️ [彈幕] 佇列已滿，丟棄最舊彈幕")
        }

        danmakuQueue.append(danmaku)
    }
}
```

#### 8.3.2 渲染優化

使用 `CADisplayLink` 同步刷新，避免過度渲染。

```swift
class DanmakuOverlay: UIView {
    private var displayLink: CADisplayLink?

    func startRendering() {
        displayLink = CADisplayLink(target: self, selector: #selector(update))
        displayLink?.add(to: .main, forMode: .common)
    }

    func stopRendering() {
        displayLink?.invalidate()
        displayLink = nil
    }

    @objc private func update() {
        // 每一幀檢查是否有新彈幕需要顯示
        processQueue()
    }

    private func processQueue() {
        // 從佇列中取出彈幕並顯示
        guard let danmaku = danmakuQueue.first else { return }

        if let trackIndex = allocateTrack(for: danmaku) {
            danmakuQueue.removeFirst()
            displayDanmaku(danmaku, in: trackIndex)
        }
        // 若無可用軌道，等待下一幀再試
    }
}
```

#### 8.3.3 記憶體優化

- 使用弱引用避免循環引用
- 及時釋放不再使用的資源
- 監聽記憶體警告並清理快取

```swift
class DanmakuManager {
    deinit {
        // 停止所有定時器
        stopRendering()

        // 清空佇列
        danmakuQueue.removeAll()

        // 清空彈幕池
        danmakuOverlay?.clearPool()

        // 移除通知觀察者
        NotificationCenter.default.removeObserver(self)
    }

    private func setupMemoryWarningObserver() {
        NotificationCenter.default.addObserver(
            self,
            selector: #selector(handleMemoryWarning),
            name: UIApplication.didReceiveMemoryWarningNotification,
            object: nil
        )
    }

    @objc private func handleMemoryWarning() {
        DXPlayerLogger.warning("⚠️ [彈幕] 收到記憶體警告，清理快取")

        danmakuOverlay?.clearPool()

        // 清空待顯示佇列（保留最近的 10 條）
        if danmakuQueue.count > 10 {
            danmakuQueue = Array(danmakuQueue.suffix(10))
        }
    }
}
```

### 8.4 時間同步機制

#### 8.4.1 播放器時間觀察

```swift
class DanmakuManager {
    private var playerTimeObserver: Any?

    private func setupPlayerTimeObserver() {
        // 每 0.1 秒觀察一次播放時間
        let interval = CMTime(seconds: 0.1, preferredTimescale: CMTimeScale(NSEC_PER_SEC))

        playerTimeObserver = mainPlayer?.addPeriodicTimeObserver(
            forInterval: interval,
            queue: .main
        ) { [weak self] time in
            let currentTime = CMTimeGetSeconds(time)
            self?.syncDanmaku(at: currentTime)
        }
    }

    private func removePlayerTimeObserver() {
        if let observer = playerTimeObserver {
            mainPlayer?.removeTimeObserver(observer)
            playerTimeObserver = nil
        }
    }

    private func syncDanmaku(at currentTime: TimeInterval) {
        // 從資料來源獲取該時間點的彈幕
        dataSource?.fetchDanmaku(from: currentTime, to: currentTime + 0.1) { [weak self] result in
            guard let self = self else { return }

            switch result {
            case .success(let danmakus):
                for danmaku in danmakus {
                    self.enqueueDanmaku(danmaku)
                }

            case .failure(let error):
                DXPlayerLogger.error("❌ [彈幕] 同步失敗: \(error.localizedDescription)")
            }
        }
    }
}
```
